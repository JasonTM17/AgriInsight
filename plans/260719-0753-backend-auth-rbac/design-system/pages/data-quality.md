# Data quality page override

Primary job: let an analyst trust, investigate, and export the current run.

- Surface freshness, completeness, validity, uniqueness, quarantine, reconciliation, run ID, cutoff date, and manifest status.
- Every issue row has an owner/recovery hint; do not hide failures behind a green aggregate badge.
- Use tables with exact column/order contracts and a visible before/after summary; preserve keyboard sort and CSV export.
- Mobile: show a severity/owner summary first, then expandable issue rows; never truncate the run/hash without an accessible full value.
